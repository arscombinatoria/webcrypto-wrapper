# AGENT Guide

このドキュメントは、このリポジトリで作業する AI エージェント向けの実行ガイドです。

## 1. Scope（適用範囲）

- 本ガイドは **リポジトリ全体** に適用する。
- より下位ディレクトリに `AGENTS.md` / `AGENT.md` がある場合は、そちらを優先する。

## 2. Purpose（目的）

- WebCrypto ラッパー実装の保守・拡張を安全に行う。
- 既存 API 互換性（`crypto-js` 互換）を維持する。
- 変更を最小限にし、テストで裏付ける。

## 3. Quick Start（最短手順）

```bash
npm ci
npm test
```

変更後に最低限実施:

```bash
npm test
```

## 4. 必須ルール

1. **後方互換性を最優先** する。
   - 既存 API の引数・戻り値・例外仕様を不用意に変更しない。
2. 変更は小さく分割し、1 コミット 1 意図を守る。
3. 実装変更時は、関連テストを追加または更新する。
4. 例外メッセージは利用者が原因を特定しやすい文言にする。
5. 不要な依存追加は避ける。
6. `import` を `try/catch` で囲まない。

## 5. 変更時チェックリスト

- [ ] 既存テスト（`__tests__/`）で期待仕様を確認した。
- [ ] 互換性破壊がないことを確認した。
- [ ] 影響範囲を最小化した。
- [ ] 新規/変更ロジックに対応するテストを追加・更新した。
- [ ] `npm test` が成功した。
- [ ] ドキュメント更新が必要なら同時に反映した。

## 6. Testing Matrix（変更種別ごとの検証）

- ドキュメントのみ変更:
  - 必須: Markdown の体裁確認
  - 推奨: `npm test`
- 実装変更:
  - 必須: `npm test`
  - 必須: 影響したテストケースの追加・更新
- API 影響が疑われる変更:
  - 必須: 既存公開 API の互換性確認
  - 必須: 例外系テスト確認

## 7. コミット規約

- コミットメッセージは「何を」「なぜ」を明確にする。
- 推奨プレフィックス: `feat:`, `fix:`, `docs:`, `refactor:`, `test:`, `chore:`

## 8. PR 本文テンプレート

以下の構成で記載する。

1. 背景
2. 変更内容
3. テスト結果（実行コマンドと結果）
4. 互換性への影響
5. リスクとロールバック方針（必要時）

## 9. Definition of Done（完了条件）

- レビュー可能な最小変更に分割されている。
- 必要なテストが追加・更新され、`npm test` が成功している。
- 互換性・例外仕様・ドキュメントの整合が取れている。
- PR 本文に、背景・変更内容・検証結果・影響範囲が明記されている。

## 10. 障害時の対応

- 失敗したテスト名と再現コマンドを記録する。
- 原因切り分けのため、変更差分を最小単位で確認する。
- 一時対応を入れる場合も、恒久対応チケット/課題を必ず残す。
